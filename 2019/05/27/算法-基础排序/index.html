就像 Hawstein 大佬说的，刷题不等于算法学习。所以还是要学算法的思想，刷题只是思想的实践。

冒泡排序

冒泡排序是基础排序之一，它基于比较来完成排序的。基本思想就是两两比较，根据大小交换元素位置。每轮冒泡确定一位有序位。所以需要 n 次冒泡，才能使整个序列有序。两个 for 循环来实现冒泡排序，外层循环控制未排序的最后位置，内层循环进行冒泡遍历。因此时间复杂度为 O(n^2)，没有使用额外的内存空间，空间复杂度为 O(1)。 因为冒泡排序只有两个元素大小不相等的时候才会进行交换，两个元素相等时，不会交换，所以冒泡排序是稳定的排序算法(大小相等的元素排序前后的相对位置是一样的)。

优化版本

冒泡排序有一个优化的地方是，当一轮冒泡结束时，没有进行过交换，则说明当前序列已经为有序序列了，可以直接结束冒泡排序。所以冒泡排序的最优时间复杂度为 O(n)。

鸡尾酒排序

鸡尾酒排序是冒泡排序的一种变体。思路是先从左到右进行一轮冒泡，再从右向左进行一轮冒泡，如此往复的进行冒泡。就像来回摇晃地调制一杯鸡尾酒，因此被称之为鸡尾酒排序。

总结就是鸡尾酒排序的平均时间复杂度为 O(n^2)，空间复杂度为 O(1)，并且为稳定的排序。在有些场景可以比冒泡排序少冒泡几轮，但在大量数据场景下由于时间复杂度的关系，该算法也是用作学习 ，不会应用。

插入排序

插入排序的思想就是把数组中的元素分成有序区间和无序区间两个区间。初始化有序区间为数组第一个元素，依次把无序区间的元素插入到有序区间合适的位置，并保证有序区间一直有序，当无序区间元素为 0 时，排序结束。因为遍历无序区间的时间复杂度为 O(n)，元素到有序区间比较并找到合适的插入位置也需要时间复杂度 O(n)，所以总的时间复杂度为 O(n^2)。最优时间复杂度的情况为，数组本来就为正序序列的时候，比较一个元素就可以判断出插入位置，最优时间度 O(n)；最坏时间复杂度的情况为，数组到倒序序列，每次查找插入位置都需要比较 n 个元素，最坏时间复杂度为 O(n^2)。没有使用额外的空间，空间复杂度为 O(1)。

选择排序

选择排序也是把序列分成两个区间，有序区间和无序区间。从无序区间中找到最小值，与有序区间后一位的元素交换位置，以此类推，直到无序区间为空，排序结束。外层循环控制无序区间的范围，内层循环遍历无序区间找到最小值，然后交换位置。
